# Flutter Module (Future) - SOP Guide

## Purpose
This folder documents the future Flutter staff module (mobile/tablet use cases).

## Non-goals (for now)
- No requirement to implement Flutter features inside current POS scope
- Flutter is future module; POS remains Next.js for now

## Expected responsibilities (future)
- Staff utilities (time-in/out, scheduling, etc.)
- Possibly staff-facing dashboards
- Must integrate with the same local-first backend contracts (no cloud)

## Constraints
- Must not require cloud sync
- Must be compatible with local-first deployment model
